# Contributing to UNReviewHub

Thank you for your interest in contributing to UNReviewHub! As a UN-managed project, we follow specific guidelines to ensure security, compliance, and code quality.

## How to Contribute

### 1. Report Bugs
Use the GitHub Issue tracker to report bugs. Please include:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Expected vs. actual behavior.
- Environment details (Browser, OS).

### 2. Suggest Enhancements
We welcome feature suggestions! Please open an issue and describe the problem you're trying to solve and how the enhancement would help.

### 3. Pull Requests
1. Fork the repository (if external).
2. Create a new branch for your feature or fix.
3. Ensure your code follows the established style (ESLint + Prettier).
4. Include tests for any new functionality.
5. Submit a PR describing your changes.

## Code Standards
- **TypeScript**: Use strict typing.
- **API**: All new endpoints must be documented in the OpenAPI spec.
- **Security**: Never commit secrets or PII. Use `.env.example` for configuration.

## Compliance & Standards
All contributions must adhere to high data protection and security standards. High-impact changes may require a thorough security review before merging.
